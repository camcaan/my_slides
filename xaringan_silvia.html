JavaScript

Markdown

A simple, in browser, Markdown driven slideshow tool targeted at people who know their way around HTML and CSS

xaringan is a package that introduces remark.js to R Markdown

CSS stands for Cascading Style Sheet, and turns

functional, but dull HTML (content)...

...into HTML (content) with style!

Remark helps us:

  • create a new slide (Markdown syntax* and slide properties);
  • format a slide (e.g. text alignment);
  • configure the slideshow;
  • and use the presentation (keyboard shortcuts).

The new directory in the "Files" pane contains

  • the new Rmd file (i.e. my-slides.Rmd)

  • the img folder containing logo files

  • the css folder containing the custom nhsr theme files
    and an HTML file in charge of adding the NHS logo to each slide

title: "Presentation title"
subtitle: "Presentation subtitle"
author: "Author Name"
institute: "NHS Organisation"
date: "Presentation Date"
output:
  xaringan::moon_reader:               # new kind of output!
    css:
     -default                          # default xaringan theme CSS file
     -css/nhsr.css                     # custom primary CSS file
     -css/nhsr-fonts.css               # custom font CSS file

More options for output xaringan::moon_reader

    lib_dir: libs                        # creates directory for libraries
    seal: false                          # false: custom title slide
    nature:
      highlightStyle: googlecode         # highlighting syntax for code
      highlightLines: true               # true: enables code line highlighting 
      highlightLanguage: ["r"]           # languages to highlight
      countIncrementalSlides: false      # false: disables counting of incremental slides
      ratio: "16:9"                      # 4:3 for standard size
    includes:
      after_body: [css/insert-logo.html] # adds NHS logo to slides (you can ignore it)

  • seal: true creates a title slide with inverse colors using metadata from your YAML

  • Other highlightStyle options from remark.js are:

    • arta, ascetic, dark, default, far, github, googlecode, idea, ir-black, magula, monokai, rainbow, solarized-dark, solarized-light, sunburst, tomorrow, tomorrow-night-blue, tomorrow-night-bright, tomorrow-night, tomorrow-night-eighties, vs, zenburn.

The nhsrtheme xaringan template comes pre-loaded with a few things

  • A setup chunk with a few libraries and some chunk options set for the whole slide deck

  • Some example slides separated by three dashes (---)

    1. Title slide with the title-slide class.
      Content is aligned with the left and the bottom of the slide
    2. Inverse slide with the inverse slide class.
      Content is aligned with middle and center of the slide
    3. Incremental slides with no specified slide class (default).
      Incremental steps are made using two dashes (--), and bullet points made using markdown (*)

Pictures

There are a variety of ways to add images to your slides!

  • Markdown
  • knitr
  • HTML
  • Image sources
  • Image paths
  • Background
![](img/camera-green.jpg)
  • simple
  • not very flexible
  • output size fully dependent on the size of the image, but scaling can be managed using custom macros (JavaScript functions)

Photo by Annie Spratt

knitr::include_graphics("img/camera-green.jpg")
  • pretty flexible
  • a little bulky
  • more options here

<img src="img/camera-green.jpg" width="90%"/>
  • most flexible
  • a bit unsightly
  • takes time to get used to syntax

Tip: To insert an avatar: 

<img src="https://insertweblink.here" width="170px" style="border-radius: 50%;"/>

Below you'll see a few places where you can find beautiful and free background images contributed by artists all over world.

Tip: You can use the .footnote[] class to credit the source of the image (example here)

  • unsplash.com
  • rawpixel.com
  • pexels.com
  • In the public domain
    (no attribution necessary)
  • Free with attribution
    (e.g. Photo by Annie Spratt)

R Markdown and xaringan don't always agree on where images are located
(i.e. relative file paths).

You might find that knitr::include_graphics() produces an image from a code chunk in your Rmd, and that the same image may not show up when you render your xaringan slides.

I recommend using this directory structure to help avoid confusion
(example file path: img/camera-green.jpg)

.
 css
 img
     camera-green.jpg
 libs
 my-slides.Rmd
 my-slides.html

You can use the background-image property underneath the --- slide separator

  • background-image:
    • url(https://insertweblink.here) --> image from the web
    • url(img/camera-green.jpg) --> local image, using relative file paths
  • background-size:
    • cover --> rescales + crops with no empty space
    • contain --> rescales only
  • background-position: play with this

background-size examples in action!

Plots

Creating side-by-side code + plot

Here's how to leverage code chunk options to avoid repeating your code:

  1. Code on left, plot on right (example)

    • Chunk 1: {r plot-right, fig.show = 'hide'}
    • Chunk 2: {r ref.label = 'plot-right', echo = FALSE}

  2. Plot on left, code on right

    • Chunk 1: {r plot-left, echo = FALSE}
    • Chunk 2: {r ref.label = 'plot-left', fig.show = 'hide'}

These and more great tips in Alison Hill's Teaching in Production

  • Bold and italic type
  • Links and images with ![]()
  • Bullet points can be added
    with -, +, or -
  • Numbered lists can be created
    with 1.

Not so familiar, from remark.js

  • First slide starts immediately after YAML, doesn't need to begin with three dashes(---)
  • Slides are separated by
    three dashes (---)
  • Incremental slides are separated by
    two dashes (--)
  • Slide content is separated from presenter notes by three questions marks (???)

There are a variety of ways to add images to your slides!

  • Markdown
  • knitr
  • HTML
  • Image sources
  • Image paths
  • Background
![](img/camera-green.jpg)
  • simple
  • not very flexible
  • output size fully dependent on the size of the image, but scaling can be managed using custom macros (JavaScript functions)

knitr::include_graphics("img/camera-green.jpg")
  • pretty flexible
  • a little bulky
  • more options here

  • most flexible
  • a bit unsightly
  • takes time to get used to syntax

R Markdown and xaringan don't always agree on where images are located
(i.e. relative file paths).

You might find that knitr::include_graphics() produces an image from a code chunk in your Rmd, and that the same image may not show up when you render your xaringan slides.

I recommend using this directory structure to help avoid confusion
(example file path: img/camera-green.jpg)

.
 css
 img
     camera-green.jpg
 libs
 my-slides.Rmd
 my-slides.html

You can use the background-image property underneath the --- slide separator

  • background-image:
    • url(https://insertweblink.here) --> image from the web
    • url(img/camera-green.jpg) --> local image, using relative file paths
  • background-size:
    • cover --> rescales + crops with no empty space
    • contain --> rescales only
  • background-position: play with this

Plots

Creating side-by-side code + plot

Here's how to leverage code chunk options to avoid repeating your code:

  1. Code on left, plot on right (example)

    • Chunk 1: {r plot-right, fig.show = 'hide'}
    • Chunk 2: {r ref.label = 'plot-right', echo = FALSE}

  2. Plot on left, code on right

    • Chunk 1: {r plot-left, echo = FALSE}
    • Chunk 2: {r ref.label = 'plot-left', fig.show = 'hide'}

These and more great tips in Alison Hill's Teaching in Production
